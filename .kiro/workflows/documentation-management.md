# Documentation Management

## Documents to Maintain

- Project Roadmap: living document tracking phases, milestones, progress
- Project Changelog: record of all significant changes, features, fixes
- System Architecture: component diagrams, data flow, integration points
- Code Standards: conventions, patterns, naming rules

## When to Update

Update documentation automatically when:

- A development phase status changes (In Progress to Complete)
- Major features are implemented or released
- Significant bugs are resolved or security patches applied
- Project timeline or scope adjustments are made
- External dependencies or breaking changes occur

## Update Protocol

1. Before: read current document status
2. During: maintain version consistency and proper formatting
3. After: verify links, dates, and cross-references are accurate
4. Quality: ensure updates align with actual implementation progress

## Changelog Format

Use Keep a Changelog format with sections: Added, Changed, Deprecated, Removed, Fixed, Security. Tag entries with semantic versioning.
